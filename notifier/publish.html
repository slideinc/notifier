notifier.publish – The Non-Coroutine Half Notifier¶

class notifier.publish.Publisher(notifier_map)¶

Publisher is a “half” notifier, meaning it can send RPCs and one-way notifications, but is unable to broadcast itself as a target of either. To provide a service in this manner, use notifier.coroutines.Notifier.

The primary reason to use a Publisher over a Notifier is that it can work outside of a gogreen coroutine environment, making it more suitable for an interactive python session, for example.

Notification Sending

publish(object, id, cmd, args)¶

Send a one-way notification to whoever is registered for receiving them with (object, id, cmd), and send args along with it.

Parameters:
  • object (str) – the service name
  • id (int) – the id to match to find a suitable recipient
  • cmd (str) – the method name on the recipient to call
  • args (tuple) – the wirebin-serializable arguments to send along with the notification

RPC Calling

rpc(object, id, cmd, args, timeout=None)¶

Send an RPC request to a registered receiver for (object, id, cmd), passing arguments tuple args. Block waiting for the response, limiting the wait to timeout seconds, if timeout is provided.

Parameters:
  • object (str) – the service name
  • id (int) – the identifier matched to find a specific handler
  • cmd (str) – the method of the service we are calling
  • args (tuple) – arguments send in the RPC request
  • timeout (int or float) – the maximum time to wait for the response
Returns:

the result returned in the RPC response
rpcs(object, id_list, cmd, args, timeout=None)¶

Sends one RPC request per id in id_list, equivalent to len(id_list) rpc() calls except when it blocks it waits on them all in parallel.

Parameters:
  • object (str) – the service name
  • id_list (list of ints) – the identifiers matched to find a specific handlers
  • cmd (str) – the method of the service we are calling
  • args (tuple) – arguments send in the RPC requests
  • timeout (int or float) – the maximum time to wait for the response
Returns:

the results returned in the RPC responses (in a list ordered the same as id_list)
rpc_call(object, id, cmd, args, source)¶

Asynchronously send an RPC request, providing an object with a callback method for when the response comes in.

Parameters:
  • object (str) – the service name
  • id (int) – the identifier matched to find a specific handler
  • cmd (str) – the method of the service we are calling
  • args (tuple) – arguments send in the RPC request
  • source –

    an object with a “rpc_response” method. that method must have the signature rpc_response(object, id, cmd, results, sequence=None)

    object, id, and cmd will be the same as were provided to the rpc_call method, results will be the result from the RPC response, and sequence may be set to an int as an identifier of which request to which it is responding.
