notifier.coroutines – The Coroutine Notifier¶

class notifier.coroutines.Notifier(notifier_map, args=())¶

Notifier is the only “full” notifier, meaning it can both send remote procedure calls and one-way notifications and announce itself as a target of them.

At creation time the Notifier needs to know about the (host, port) of all other notifiers with which it will communicate, so notifier_map is a list of those 2-tuples. args will be passed on to the run() method when the Notifier’s coroutine thread is started.

run(smap, port=0, host='')¶

Being a gogreen.coro.Thread subclass, the Notifier.run arguments must be filled ahead of time by args and kwargs keyword arguments to Notifier.__init__.

Parameters:
  • smap (list) – a list of (host, port) pairs with the locations of all the other Notifiers on the network.
  • port (int) – the port the Notifier will bind to
  • host (str) – the host the Notifier will bind to

One-Way Notification Registration Management

subscribe(object, id, cmd, destination)¶

Register for receiving one-way notifications for this set of identifiers.

Parameters:
  • object (str) – the service name for which we are subscribing to messages
  • id (tuple) – a (mask, value) pair to match ids
  • cmd (str) – the method of the destination to call
  • destination – the object from which we grab the cmd method to handle the request.
slice(object, id, cmd, destination, weight=1.0)¶

Register for receiving one-way notifications for this set of identifiers, adjusting the weight used for this notifier when the notification is sent to a random recipient.

Parameters:
  • object (str) – the service name for which we are subscribing to messages
  • id (tuple) – a (mask, value) pair to match ids
  • cmd (str) – the method of the destination to call
  • destination – the object from which we grab the cmd method to handle the request.
  • weight (float) – the weight relative to the other nodes registered for this event for random sends.
unsubscribe(object, id, cmd, destination)¶

Unregister from receiving notifications to object/id

Parameters:
  • object (str) – the service name from which we are unsubscribing
  • id (tuple) – the (mask, value) for which we will no longer receive
  • cmd (str) – the method of the destination to call
  • destination – the object from which we would have grabbed the cmd method to handle the request.
unregister_all(destination)¶

Unregister from receiving all notifications for which the notifier is currently subscribed and delgating to destination.

Parameters:
  • destination – the object from which we would have grabbed the cmd method to handle the request.

Notification Sending

publish(object, id, cmd, args)¶

Send a one-way notification to whoever is registered for receiving them with (object, id, cmd), and send args along with it.

Parameters:
  • object (str) – the service name
  • id (int) – the id to match to find a suitable recipient
  • cmd (str) – the method name on the recipient to call
  • args (tuple) – the wirebin-serializable arguments to send along with the notification

Register for receiving RPC requests for this set of identifiers.

Parameters:
  • object (str) – the service name for which we are subscribing
  • id (tuple) – a (mask, value) pair to match ids
  • cmd (str) – the method of the destination call

rpc_slice(object, id, cmd, destination, weight=1.0)¶

Register for receiving RPC requests for this set of identifiers, and adjust the weight for this notifier when a random destination is selected.

Parameters:
  • object (str) – the service name for which we are subscribing
  • id (tuple) – a (mask, value) pair to match ids
  • cmd (str) – the method of the destination call
  • destination – the object from which the notifier will getattr() the cmd to get the function to use to handle the RPC request.
  • weight (float) – the weight (normal is 1.0) for this notifier in calls to this (object, id, cmd) on the network

rpc_unregister_all(destination)¶

RPC Calling

rpc(object, id, cmd, args, timeout=None)¶

Send an RPC request to a registered receiver for (object, id, cmd), passing arguments tuple args. Block waiting for the response, limiting the wait to timeout seconds, if timeout is provided.

Parameters:
  • object (str) – the service name
  • id (int) – the identifier matched to find a specific handler
  • cmd (str) – the method of the service we are calling
  • args (tuple) – arguments send in the RPC request
  • timeout (int or float) – the maximum time to wait for the response
Returns:

the result returned in the RPC response
rpcs(object, id_list, cmd, args, timeout=None)¶

Sends one RPC request per id in id_list, equivalent to len(id_list) rpc() calls except when it blocks it waits on them all in parallel.

Parameters:
  • object (str) – the service name
  • id_list (list of ints) – the identifiers matched to find a specific handlers
  • cmd (str) – the method of the service we are calling
  • args (tuple) – arguments send in the RPC requests
  • timeout (int or float) – the maximum time to wait for the response
Returns:

the results returned in the RPC responses (in a list ordered the same as id_list)
rpc_call(object, id, cmd, args, source)¶

Asynchronously send an RPC request, providing an object with a callback method for when the response comes in.

Parameters:
  • object (str) – the service name
  • id (int) – the identifier matched to find a specific handler
  • cmd (str) – the method of the service we are calling
  • args (tuple) – arguments send in the RPC request
  • source –

    an object with a “rpc_response” method. that method must have the signature rpc_response(object, id, cmd, results, sequence=None)

    object, id, and cmd will be the same as were provided to the rpc_call method, results will be the result from the RPC response, and sequence may be set to an int as an identifier of which request to which it is responding.
